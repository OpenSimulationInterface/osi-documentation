struct osi3::LaneBoundary::Classification

Classification of a lane boundary.

Note

In the example, the symbols l1, l2, … and lb1, lb2, … stand for the lane ids and lane boundary ids respectively, i.e. for integers (uint64). The symbols cl1, cl2, … represent the osi3::Lane::Classification::centerline elements of the lanes with the respective ids.

Public Types

enum Type

The lane boundary type. There is no special representation for double lines, e.g. solid / solid or dashed / solid. In such cases, each lane will define its own side of the lane boundary.

Values:

enumerator TYPE_UNKNOWN = 0

The type of lane boundary is unknown. Value must not be used in ground truth.

enumerator TYPE_OTHER = 1

Unspecified but known type of lane boundary. Consider proposing an additional type if using TYPE_OTHER.

enumerator TYPE_NO_LINE = 2

An invisible lane boundary (e.g. unmarked part of a dashed line).

enumerator TYPE_SOLID_LINE = 3

A solid line at the lane boundary.

enumerator TYPE_DASHED_LINE = 4

A dashed line at the lane boundary.

enumerator TYPE_BOTTS_DOTS = 5

A lane boundary consisting of Botts’ dots (multiple Botts dots).

enumerator TYPE_ROAD_EDGE = 6

A lane boundary formed by the road’s edge. The road edge is the end of the (paved) road surface.

enumerator TYPE_SNOW_EDGE = 7

A lane boundary formed by a snow edge that may be on the road surface.

enumerator TYPE_GRASS_EDGE = 8

A lane boundary covered by grass.

enumerator TYPE_GRAVEL_EDGE = 9

A lane boundary covered by gravel.

enumerator TYPE_SOIL_EDGE = 10

A lane boundary covered by soil.

enumerator TYPE_GUARD_RAIL = 11

A guard rail.

enumerator TYPE_CURB = 12

A curb.

enumerator TYPE_STRUCTURE = 13

A structure (e.g. building or tunnel wall).

enum Color

The color of the lane boundary in case of a lane markings. Lane markings that alternate in color must be represented by individual LaneBoundary segments.

Values:

enumerator COLOR_UNKNOWN = 0

Color of marking is unknown. Value must not be used in ground truth.

enumerator COLOR_OTHER = 1

Other (unspecified but known) color.

enumerator COLOR_NONE = 2

Marking without color. Used to represent logical boundaries without actual physical markings at the respective position. Value may be used in ground truth only.

enumerator COLOR_WHITE = 3

Marking with white color.

enumerator COLOR_YELLOW = 4

Marking with yellow / orange-yellow color.

enumerator COLOR_RED = 5

Marking with red color

enumerator COLOR_BLUE = 6

Marking with blue color.

enumerator COLOR_GREEN = 7

Marking with green color;

enumerator COLOR_VIOLET = 8

Marking with violet color.

Public Members

optional Type type = 1

The ids of StationaryObject which limit the corresponding lane. This field must be set if the type is set to TYPE_STRUCTURE
